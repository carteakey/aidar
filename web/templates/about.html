aidar tracks stylistic patterns across the web that correlate with AI-era writing. It doesn't know if you're a robot. It just notices things.

what it measures

Known linguistic signals: em dash frequency, hedging phrases (“it’s worth noting”, “delve into”, “key takeaway”), bullet point density, header overuse, formal register vocabulary, emoji density. Each site gets a Stylistic Index from 0–100. Higher = more signal. Lower = less signal. Draw your own conclusions.

what it is not

Not a classifier. Not a detector. Not a judgment. Humans use em dashes. AI writes without them. The index is a data point for studying how the web’s writing style is shifting—not a verdict on any individual page or author.

the irony

This tool was written by Claude (an AI). The patterns it detects are largely patterns Claude itself exhibits. We are aware of this. We find it funny. The em dashes in the source code are very real.

inspired by

New accounts on Hacker News ten times more likely to use em-dashes — Marginalia, Feb 2026.

source

github.com/carteakey/aidar  —  open source, MIT licensed, patterns are YAML files you can contribute to.

stack

Python · FastAPI · SQLite · Litestream (continuous replication to Cloudflare R2) · trafilatura (scraping & sitemap discovery) · Heroku · patterns in YAML. No Postgres. No ORM. No vector DB. Just a SQLite file and a cron job.
